Description

Machine learning models are used to detect potentially fraudulent online actions such as online payment fraud and the creation of fake accounts using past fraud data. Each model is trained with a model type, which is a specialized recipe to build a fraud detection model for a specified fraud use case. Deployed models are imported into detectors, where you can configure decision logic (for example, rules) to interpret the model's score and assign outcomes like pass or submit transaction to a human investigator for review.

The overall workflow of getting fraud predictions include defining the event needed for evaluation, gathering historical event data such as training data, creating a trained model using a model type and creating a detector that includes the model version and decision rules. A fraud detection will be generated once, and event is sent to the Fraud Detector. AWS Console or AWS Command Line Interface can be used for creating and managing models and detector versions.
